# CNIPA 单文档重排修复规范（R46）

- 时间戳：2026-02-14 21:35:00 +08:00
- 执行目标：修复 `patent/说明书_资源调度_CNIPA_完整稿_R44.md` 的拼接结构错误，并补强 `patent/权利要求书_资源调度_CNIPA_修正版_R44.md` 中从属权利要求 5/6/7 的技术边界。

## 1. 问题定义

R44 版本存在以下结构性缺陷：
1. 说明书存在两套并列结构（前置壳 + v3 整体粘贴），导致重复的“技术领域/背景技术/摘要”。
2. 从属权利要求 5/6/7 仅有一句话，缺少可执行技术特征。
3. 附图当前为实验阶段图面，存在英文标注与参考编号不完整问题，需在正式文本中明确提交前规范化要求。

## 2. 修复策略

### 2.1 说明书重排规则（单文档）

1. 保留 CNIPA 六章主结构：
   - 一、发明名称
   - 二、技术领域
   - 三、背景技术
   - 四、发明内容
   - 五、附图说明
   - 六、具体实施方式
2. 内容映射规则：
   - `v3 §2.1 + §2.2 + §2.3` -> `三、背景技术`
   - `v3 §3.1 + §3.2 + §3.3` -> `四、发明内容`
   - `v3 §5~§20` -> `六、具体实施方式`
   - `v3 §4` -> `五、附图说明`
3. 不得再出现第二套 `## 一、技术领域` 等顶层标题。

### 2.2 权利要求 5/6/7 补强规则

1. 权利要求 5（阻断审计事件）必须写明记录字段，至少包括：任务标识、阻断原因类别、时间戳、统计计数。
2. 权利要求 6（终止失败保护）必须写明条件与动作：超时未终止 -> 强制移除 -> 记录终止失败/超时移除事件。
3. 权利要求 7（自适应参数调优）必须写明调优输入与输出：重试原因类别 -> 调整阈值偏置、调度窗口、负载参数。

### 2.3 附图规范化声明

在说明书“附图说明”章节新增正式声明：
1. 当前图 1~4 为实验阶段生成图。
2. 正式提交前统一中文标注。
3. 正式提交前补齐参考编号并与附图标记说明表一一对应。

## 3. 验证要求

1. 文档完整性：`patent/说明书_资源调度_CNIPA_完整稿_R44.md` 不再包含分隔线拼接痕迹。
2. 结构一致性：文档顶层仅保留六章。
3. 权利要求可执行性：5/6/7 均含具体技术动作与判断条件。
4. 三板斧通过：结构检查、配置校验、全量测试。

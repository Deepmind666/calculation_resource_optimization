# CNIPA Formal Document Profile (R43)

- Timestamp: 2026-02-14 18:17:31 +08:00
- Purpose: define a consistent CNIPA-style drafting profile for the current resource-scheduler patent set
- Scope: document structure and wording conventions only

## 1. Required CNIPA File Set

1. Claim file: 权利要求书
2. Specification file: 说明书
3. Abstract file: 摘要
4. Figure brief file: 附图说明
5. Figure text draft for drafter: 附图文字描述稿
6. Reference numeral table: 附图标记说明表

## 2. Section Order for Specification

1. 发明名称
2. 技术领域
3. 背景技术
4. 发明内容
5. 附图说明
6. 具体实施方式

## 3. Claim Wording Rules

1. Independent claim starts with: “一种……，其特征在于……”
2. Dependent claim starts with: “根据权利要求X所述……，其特征在于……”
3. Avoid code-level terms and variable names.
4. Keep method/system/media three-layer structure.

## 4. Figure Text Rules

1. Each figure must have one formal CNIPA sentence in “附图说明”.
2. Each figure must have 5-10 lines of textual drawing instructions in “附图文字描述稿”.
3. Reference numerals must be consistent with the reference numeral table.

## 5. Current Round Objective

1. Convert current scheduler patent draft into CNIPA-style formatted text artifacts.
2. Provide a figure text draft that can be handed to drawing/agent teams directly.

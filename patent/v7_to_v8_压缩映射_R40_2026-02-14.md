# v7 到 v8 压缩映射（R40）

- 时间戳：2026-02-14 15:30:32 +08:00
- 目标：说明 v8 如何在保留核心链路前提下压缩独立权利要求长度

## 1. 变更原则

1. 保留创造性主链：双视图判定 -> 同周期累计投影 -> 紧急归一化回收 -> 双目标停止。
2. 将工程细节从独立权利要求下沉到从属权利要求，降低独立条款冗长度。
3. 保留方法/系统/介质三层结构，减少跨层重复表述。

## 2. 独立权利要求压缩点

| v7 位置 | v8 位置 | 压缩方式 | 影响 |
|---|---|---|---|
| 方法独立权利要求1第3步（绑定/未绑定详细规则） | 权利要求2 | 从独立条款下沉到从属条款 | 独立条款长度下降 |
| 方法独立权利要求1第4步（协同增强项细节） | 权利要求3 | 保留主意图，下沉触发细节 | 核心链路保持 |
| 方法独立权利要求1第5步（亲和梯度细节） | 权利要求4 | 梯度关系下沉为从属条款 | 独立条款更紧凑 |
| v7 权利要求5/6/7/8/9 的部分工程稳定性条款 | 权利要求5 | 合并为“附加机制集合” | 条款总数压缩 |
| 系统类权利要求10/11 | 权利要求6/7 | 合并模块描述，去重字段 | 系统层更简洁 |

## 3. 条款数量变化

1. v7：12 条（方法 1-9，系统 10-11，介质 12）。
2. v8：8 条（方法 1-5，系统 6-7，介质 8）。
3. 压缩比例：总条款数下降约 33.3%。

## 4. 风险提示

1. v8 压缩后，从属条款颗粒度更粗，部分实施细节保护力度低于 v7。
2. 若审查员要求进一步限定，建议优先回补 v7 中被下沉条款。

## 5. 使用建议

1. 首轮答审优先 v8（便于阅读和快速沟通）。
2. 二轮答审或范围回补时，使用 v7 对应条款进行定向增强。
